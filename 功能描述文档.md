 
## 0. 产品定位

 面向**学生、家长、老师**的学习陪伴工具。学生用 iPhone 拍题或开启智能连拍记录一段学习过程，系统记录画面、回答学习问题、整理错题、生成报告、给出复习建议。

它**不是**单纯的"拍照搜题"，而是帮助用户看清一段学习过程：学了什么、哪里卡住、问了什么、哪些题可能错了、后面应该怎么复习。
 

 

总结：

过程自动记录，过程实时帮助，结果异步生成

一，过程自动记录

1，打开手机，自动开启 观察模式，只有在设置中才能手动关闭。

学生把 iPhone 放在桌面旁开启连拍。系统持续观察画面，**只记录有价值的变化帧**：
- 题目进入画面 / 学生开始书写 / 翻页 / 答案变化 / 出现订正 / 长时间停留某题

空桌面、随手乱放、模糊、遮挡严重、无学习材料 → **自动忽略，不写进报告**，也不啰嗦提示用户。

停止连拍后生成本次学习报告。

app UI需要有一个小窗口，可以预览当前的画面，点击放大，长按关闭，关闭后会出现一行小字，显示当前相机的日志，比如，观察中：未看到学习材料/捕获变化帧/缓存已提交/ 等等

---

二，过程实时帮助

 实时语音问答（QA）
 

两种打开麦克风的方式：
1，点击屏幕上的语音输入按钮，
2，观察模式识别到食指指向一个题目，就会自动打开麦克风。


例如：学生直接开口问，
- "这道题怎么做？"
- "第二题我哪里错了？"
- "为什么这里要这样算？"
- "帮我检查一下对不对。"
- "换一种简单的方法讲。"

系统**结合当前画面和最近学习上下文**回答。普通追问沿用上一道题上下文，不要求每次重新拍照。

触发方式：
- 点击语音提问
- 食指指题
- OK 手势追问
- Web 端远程触发
V手势结束本轮问答

行为约定：
- 识别结束后冻结问题文本，避免提交空问题
- 没听清不触发 AI；沉默自动结束
- 当前帧抓拍等待约 4 秒，失败立即降级，不等完整超时
- 抓拍失败回退到拍照
- 答案用中文语音朗读，追问/结束/相机失败时及时停止朗读

**原则**：看不清时提示重新拍摄，**不编造题目内容**。

 
 手势交互

| 手势 | 含义 |
|---|---|
| 食指 | 指题并启动问答 |
| OK / 点赞 | 追问 / 打断当前朗读 |
| V / 结束 | 结束当前问答 |
 

高风险动作（如结束整轮学习）不能只靠单帧手势执行。

  AI 语音朗读要求

- 回答短、清楚、适合学生理解
- 长答案优先讲思路和关键步骤
- 用户打断时及时停止
- 语音播放失败时文字答案仍可看


三，结果异步生成


  学习报告

学习结束后生成本次报告，包含：
- 本次学习大致内容
- 拍到的题目和作答情况
- 可能的错题
- 相关知识点
- 学生问过的问题
- 需要复习的内容
- 给家长/老师的后续建议

**报告必须基于实际拍到的证据**。没拍到、看不清、无法判断的内容**不能写成确定结论**。

 学习清单

把本次学习中识别到的内容整理成清单：
- 做了哪些题
- 涉及哪些知识点
- 哪里有作答或订正
- 哪些内容值得回看

 错题候选

把疑似错题整理出来供家长/老师/管理员确认。错题状态流转：

| 状态 | 含义 |
|---|---|
| 疑似错题 | 抽取规则初筛 |
| 确认错题 | 人工确认 |
| 已忽略 | 不是真错题 |
| 已订正 | 已改正 |
| 已掌握 | 学生掌握 |

每条错题包含：科目、页码、题号、位置、错误类型、订正、下一步动作、来源证据图。

  复习队列与今日复习

确认后的错题进入复习队列。复习时记录：

| result | 含义 |
|---|---|
| right | 复习正确 |
| wrong | 复习错误 |
| later | 延后复习 |
| mastered | 已掌握 |

系统根据结果安排下次复习时间。

**今日复习入口**：学生打开 iPhone 点"今日复习"，系统自动读取复习队列选到期错题，带着题目、位置、错因、知识点和下一步建议进入 AI 复习问答。
- 无到期错题时可提前复习一条
- 完全无错题时降级为 5 分钟复习计划
- 复习回合不抓当前镜头，避免桌面内容污染错题上下文

 长期学习画像

根据多次学习记录汇总：
- 常见薄弱知识点
- 常见错误类型
- 科目分布
- 最近学习内容
- 复习情况
- 哪些问题经常被追问
